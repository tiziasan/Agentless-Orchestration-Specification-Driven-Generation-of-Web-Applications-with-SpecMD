# Agentless-Orchestration-Specification-Driven-Generation-of-Web-Applications-with-SpecMD

Supplemental material and replication package for the paper *Agentless Orchestration: Specification-Driven Generation of Web Applications with SpecMD*.

This repository contains the SpecMD specification template, the SpecMD Builder tool, the source of every generated web application, the human evaluation material, and the full results for RQ1, RQ2, and RQ3.

## Repository contents

- `specmd-builder/` SpecMD Builder, the browser tool that compiles a SpecMD specification into a runnable TypeScript project.
- `template/` The SpecMD Markdown specification template and a worked example (StockBrief).
- `applications/specmd/` The 101 web applications generated by SpecMD Builder.
- `applications/webgen-lm-32b/` The matching web applications generated by WebGen-LM-32B.
- `screenshots/` Side-by-side screenshots of each evaluated application pair.
- `results/` Full results for RQ1 (functional benchmark), RQ2 (human evaluation), and RQ3 (code characterization).
- `survey/` The survey instrument used in the human evaluation.
- `demographics/` Demographic data of the four template experts and the 21 study participants.

## Running the SpecMD Builder tool

SpecMD Builder is a single-file React application that runs entirely in the browser. No build step or server is required.

Open `specmd-builder/index.html` in a web browser.

From there you can author a specification by hand or infer it from a free-text prompt, then generate and download the application as a ZIP.

## Running a generated web application

Each generated application is a standard React 18 + Vite 5 + TypeScript 5 project. To run one, enter its folder and execute:

```bash
npm install
npm run dev
```

Then open the local URL printed by Vite in your browser.

## Results

The `results/` folder contains the data behind the tables and figures in the paper:

- RQ1: per case and per category outcomes on the 647 WebGen-Bench test cases.
- RQ2: the 198 paired judgments, per dimension preference rates, and per application majority votes.
- RQ3: token usage, lines of code, dependency counts, and SonarQube findings across the 101 applications.

## Screenshots

The `screenshots/` folder contains the side-by-side captures shown to evaluators, with the SpecMD Builder and WebGen-LM-32B applications presented under neutral labels for each instruction.
